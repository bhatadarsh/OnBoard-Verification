To handle special characters and non-string values, I ensure my validation script strictly enforced proper coding and typecasting. For special characters, dollar or exclamation, I used Base64 encoding during the storage and injection process to prevent shell injection or premature expansion. Recording them only at runtime. For non-string values, I implemented schema validation using JSON schema or pydent, which explicitly requires variables to be cast to the expected type, such as integers or booleans, before the requirement process. Ensuring that any type mismatch triggers an immediate pipeline failure rather than a runtime error.
