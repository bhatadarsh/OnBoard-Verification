In the previous project, I was working on a data pipeline where a Python-based processing was taking too long to transform and load large datasets, causing a bottleneck in our daily reporting. So the actions which I took were the profiling. First, I didn't guess the problem. I used cprofile module to identify exactly what's the problem. And vectorization, instead of filtering through rows, I preferred the code to use the vectorized operations. And for multiprocessing, for the last, it was CPU-born, but independently, I used a multiprocessing library. And then for data annealing, I switched from reading entire service point to memory to using things from more efficient formats like Paracate, which reduced our memory, footprint, and I.O. wait times.
